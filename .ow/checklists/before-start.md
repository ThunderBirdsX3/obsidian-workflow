# Checklist — Before Start

ก่อนเริ่มทำงานใดๆ ตรวจ:

- [ ] อ่าน `docs/vault/00-Index/IMPLEMENTATION-STATUS.md`
- [ ] อ่าน task description / issue / plan file
- [ ] ระบุ success criteria ที่ตรวจสอบได้จริง (ก่อนลงมือ)
- [ ] ระบุ scope: เฉพาะอะไร? ไม่ทำอะไร?
- [ ] ระบุ area (backend / frontend / mobile / docs)
- [ ] อ่าน vault docs ที่เกี่ยวข้อง (PRD, SRS, Feature, Function)
- [ ] ตรวจ in-progress plans/fixes อื่นที่อาจ conflict
- [ ] ระบุ assumption ที่ใช้ (ถ้ามี)
- [ ] ตรวจว่า branch ถูก (main / develop / feature)
- [ ] ตรวจว่า working tree clean (no uncommitted changes ที่ไม่เกี่ยว)
